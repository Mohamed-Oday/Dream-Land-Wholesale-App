<objective>
## Goal
Driver can create an order at a store (select store, add products with quantities, review totals) and see an on-screen receipt preview. Owner can view all orders. This completes the Phase 1 "core loop" — a driver takes an order and the owner sees it.

## Purpose
This is the "it works" moment for the business. Until now the app has auth, products, and stores — but no order flow. Plan 01-04 proves the end-to-end value proposition: driver creates order in the field → owner sees it in real-time via Supabase.

## Output
- `lib/features/orders/` — repository, providers, and 3 screens
- Modified driver shell (Orders tab wired up) and owner shell (orders accessible)
- Updated l10n with order-related Arabic strings
</objective>

## Supabase Schema Reference (from migration)
orders: id(uuid), business_id(uuid), store_id(uuid), driver_id(uuid), subtotal(numeric),
  tax_percentage(numeric), tax_amount(numeric), discount(numeric), discount_status(text),
  discount_approved_by(uuid?), total(numeric), status(text: created|delivered|cancelled), created_at(timestamptz)

order_lines: id(uuid), order_id(uuid), product_id(uuid), quantity(int), unit_price(numeric),
  packages_count(int?), line_total(numeric)

RLS: drivers insert own orders (driver_id = auth.uid()), select own orders. Owners select all.

## UX Design Specifications (from ui-ux-pro-max)

### Order List Screen
- ConsumerWidget watching FutureProvider (established pattern)
- Each order card: store name (bold), date (secondary), total with DA currency, status chip
- Status chips: created=orange(primary), delivered=green(success), cancelled=red(error)
- Empty state with illustration text + "Create your first order" message
- FAB for new order creation
- Pull-to-refresh with ref.invalidate()

### Create Order Screen
- Single scrollable screen (NOT multi-step wizard — simpler for field workers)
- Section 1 — Store selector: DropdownButtonFormField populated from storeListProvider
- Section 2 — Line items: "Add Product" button opens ModalBottomSheet with product catalog
  - Each product in sheet shows name + price, tap adds it to order
  - Added items display as Cards with: product name, unit_price, quantity +/- controls, line_total
  - Swipe-to-dismiss or delete icon to remove items
  - Minimum quantity = 1, no upper bound
- Section 3 — Totals: subtotal, tax_amount (auto-calc from tax_percentage), total
  - tax_percentage defaults to 0 (owner can configure later)
  - discount = 0, discount_status = 'none' (Phase 3 feature)
- Bottom: "Create Order" FilledButton (disabled until store + at least 1 item selected)
  - Shows loading spinner during async save
  - On success → navigate to ReceiptPreviewScreen with order data
- Touch targets: all buttons/controls >= 48dp height
- 16dp horizontal padding, 8dp spacing rhythm

### Receipt Preview Screen
- Receipt-style card layout with white background
- Header: store name + address, order date/time
- Divider
- Line items table: product name | qty | unit_price | line_total (tabular figures for alignment)
- Divider
- Totals section: subtotal, tax (if > 0), total (bold, larger size)
- Status badge at top
- Bottom buttons: "Done" (returns to order list), "Print" (disabled/placeholder — Phase 2)
- Also serves as order detail view when tapping existing orders in list
- RTL layout with Arabic number formatting (DA currency)

## AC-1: Order List Displays Driver's Orders
```gherkin
Given a driver is logged in and has existing orders
When they navigate to the Orders tab
Then they see a reverse-chronological list of their orders
  showing store name, date, total amount (DA), and status for each
```

## AC-2: Empty Order State
```gherkin
Given a driver is logged in with no orders
When they navigate to the Orders tab
Then they see an empty state message encouraging them to create their first order
  and a FAB button to start
```

## AC-3: Create Order — Store Selection
```gherkin
Given a driver taps the FAB on the order list
When the create order screen loads
Then they see a dropdown of available stores
  and cannot proceed without selecting one
```

## AC-4: Create Order — Add Line Items
```gherkin
Given a driver has selected a store on the create order screen
When they tap "Add Product"
Then a bottom sheet shows the product catalog
  and tapping a product adds it to the order with quantity 1
  and they can adjust quantity with +/- controls
  and they can remove items
```

## AC-5: Create Order — Calculations
```gherkin
Given a driver has added products to an order
When quantities change
Then line_total = quantity * unit_price for each item
  and subtotal = sum of all line_totals
  and total = subtotal + tax_amount - discount
  and all values update in real-time
```

## AC-6: Create Order — Submit
```gherkin
Given a driver has selected a store and added at least one product
When they tap "Create Order"
Then the order and its line items are saved to Supabase
  and the button shows loading state during save
  and on success, the driver is navigated to the receipt preview
```

## AC-7: Receipt Preview Displays Order Details
```gherkin
Given an order was just created or a driver taps an existing order
When the receipt preview screen loads
Then it shows: store name, date, line items (product, qty, price, total per line),
  subtotal, tax (if > 0), total, and status
```

## AC-8: Owner Views All Orders
```gherkin
Given an owner is logged in
When they navigate to orders
Then they see all orders from all drivers
  with store name, driver name, date, total, and status
```
