<objective>
## Goal
Hook push notifications into all 6 business events and add per-user notification preferences with a settings UI, completing the real-time visibility loop.

## Purpose
Owner/admin phones ring when drivers take actions in the field — the final piece of v0.3 before release. Without triggers, the FCM infrastructure from Plan 12-01 sits idle.

## Output
- `sendNotification()` helper in NotificationService (centralizes Edge Function calls)
- 6 event triggers in 4 screens (order, payment, shift open, shift close)
- Low stock check after order creation
- `notification_preferences` table with per-event boolean toggles
- Updated `get_fcm_tokens_for_business` RPC to filter by preferences
- Notification preferences screen in settings with 6 toggle switches
</objective>

<acceptance_criteria>

## AC-1: Order Notification
```gherkin
Given a driver creates an order successfully
When the order is saved via create_order_atomic RPC
Then a "new_order" notification is sent to owner/admin devices (excluding the caller)
And the notification contains the driver's name and store name
```

## AC-2: Discount Pending Notification
```gherkin
Given a driver creates an order with a discount (discountStatus = 'pending')
When the order is saved
Then a "discount_pending" notification is also sent to the owner only
And the notification contains the driver's name, store name, and discount amount
```

## AC-3: Payment Notification
```gherkin
Given a driver collects a payment from a store
When the payment is saved via create_payment RPC
Then a "payment_collected" notification is sent to owner/admin devices
And the notification contains the driver's name, amount collected, and store name
```

## AC-4: Low Stock Notification
```gherkin
Given products in an order have stock_on_hand <= low_stock_threshold after deduction
When the order completes successfully
Then a "low_stock" notification is sent for each affected product
And the notification contains the product name and remaining quantity
```

## AC-5: Shift Opened Notification
```gherkin
Given an admin/owner creates a driver load (opens a shift)
When the load is saved via create_driver_load RPC
Then a "shift_opened" notification is sent to owner/admin devices
And the notification contains the driver's name
```

## AC-6: Shift Closed Notification
```gherkin
Given a driver closes their shift
When the load is closed via close_driver_load RPC
Then a "shift_closed" notification is sent to owner/admin devices
And the notification contains the driver's name
```

## AC-7: Notification Preferences Filtering
```gherkin
Given a user has disabled "new_order" in their notification preferences
When another user triggers a new_order event
Then the user with the disabled preference does NOT receive the notification
And users with default preferences (no row) still receive it
```

## AC-8: Preferences UI
```gherkin
Given a user navigates to Settings → Notifications
When the notification preferences screen loads
Then 6 toggle switches are displayed (one per event type), all defaulting to ON
And toggling a switch immediately persists the preference via RPC
```

## AC-10: Preferences Screen Error Handling <!-- audit-added -->
```gherkin
Given the device is offline or the RPC fails when loading preferences
When the notification preferences screen attempts to fetch preferences
Then an error state is shown with a retry button (not a crash or empty screen)
And toggling a switch while offline shows an error snackbar and reverts the toggle
```

## AC-11: Preferences Visibility by Role <!-- audit-added -->
```gherkin
Given a user with role "driver" is on the Settings screen
When the settings list renders
Then the Notifications card is NOT shown (drivers don't receive notifications)
And only owner and admin roles see the Notifications navigation
```

## AC-9: Best-Effort Non-Blocking
```gherkin
Given the Edge Function is unavailable or returns an error
When a business event triggers a notification
Then the core operation (order, payment, shift) still succeeds normally
And the notification failure is logged via debugPrint only
```
