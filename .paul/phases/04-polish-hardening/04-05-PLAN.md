<objective>
## Goal
Harden offline sync (cleanup + periodic processing + status visibility), add printer auto-reconnect on print failure, and set up DriverLocation 7-day data retention.

## Purpose
The sync queue accumulates synced items indefinitely and has no automatic processing. Printer disconnections during field use require manual reconnection before reprinting. DriverLocation records grow unbounded (append-only GPS pings every 30s). These are the final hardening items for production field use.

## Output
- Updated app color palette (brand palette from design spec)
- Sync queue cleanup (delete synced items) + pending count on settings
- PrintService auto-reconnect attempt before print failure
- Supabase function for DriverLocation 7-day retention cleanup
- 3 new l10n strings (AR + EN)
</objective>

    **Part B — DriverLocation 7-day retention:**

    Create `supabase/migrations/010_location_retention.sql`:
    ```sql
    -- Cleanup function: delete driver_locations older than 7 days
    CREATE OR REPLACE FUNCTION cleanup_old_driver_locations()
    RETURNS INTEGER AS $$
    DECLARE
      v_count INTEGER;
    BEGIN
      DELETE FROM driver_locations
      WHERE created_at < NOW() - INTERVAL '7 days';
      GET DIAGNOSTICS v_count = ROW_COUNT;
      RETURN v_count;
    END;
    $$ LANGUAGE plpgsql SECURITY DEFINER;

    GRANT EXECUTE ON FUNCTION cleanup_old_driver_locations() TO authenticated;
    ```

    This function can be:
    - Called manually via Supabase dashboard SQL editor
    - Scheduled via pg_cron extension (configured in Supabase dashboard)
    - Called from the app as an owner action (future enhancement)

    Avoid: Adding pg_cron setup in the migration (requires Supabase dashboard configuration, not SQL migration). Avoid calling cleanup on every app start (unnecessary load).
