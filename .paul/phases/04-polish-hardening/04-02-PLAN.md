<objective>
## Goal
Add date range filtering to all list/history views (orders, payments, packages) and create a driver performance view showing per-driver order/payment stats.

## Purpose
Currently all list screens show ALL records with no time filtering. For a business processing 10-20 orders daily, this list grows unwieldy fast. The owner also has no way to evaluate individual driver performance — how many orders a driver handled, how much they collected, which stores they visited.

## Output
- Reusable DateRangeFilterBar widget with preset chips (Today, This Week, This Month, All)
- Date range filtering on 3 list screens (orders, payments, packages)
- Driver performance screen with order/payment stats and activity timeline
- 10 new l10n strings (AR + EN)
</objective>

## AC-1: Date Range Filter Bar on List Screens
```gherkin
Given any list screen (orders, payments, packages)
When the screen loads
Then a horizontal chip row is shown below the AppBar with presets: Today, This Week, This Month, All
And "Today" is selected by default
And selecting a different preset filters the list to that date range
And the list refreshes immediately showing only records within the range
```

## AC-2: Supabase Date Filtering
```gherkin
Given a date range is selected on a list screen
When the provider fetches data from the repository
Then the Supabase query includes .gte('created_at', startDate) and .lte('created_at', endDate) filters
And only records within the date range are returned
And "All" preset removes date filters (shows all records)
```

## AC-3: Driver Performance View
```gherkin
Given the owner or admin is on the user management screen
When they tap a driver's list tile
Then a DriverPerformanceScreen opens showing:
  - Driver name and role
  - Total orders count and total order value
  - Total payments collected count and total payment amount
  - Recent activity timeline (last 20 orders + payments, interleaved by date)
```

End = DateTime.now().toUtc().** <!-- audit-added: end must be current moment, not start-of-day, otherwise .lte() misses all same-day records -->
    - Helper function `thisWeekRange()` — Monday midnight Algeria (UTC) to DateTime.now().toUtc()

    Create `lib/core/widgets/date_range_filter_bar.dart`:
