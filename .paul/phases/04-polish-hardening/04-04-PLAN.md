<objective>
## Goal
Add in-app update check via remote config and apply targeted performance optimizations to reduce UI jank.

## Purpose
The app is distributed via APK sideload with no Play Store auto-update. Users have no way to know a new version is available. The owner reported ~30fps lag — basic widget tree optimizations (RepaintBoundary on list items, const constructors) will improve perceived smoothness.

## Output
- Remote config table in Supabase with latest_version + download_url
- Version check on settings screen with update prompt
- Current version display in settings
- RepaintBoundary on list card widgets across 3 list screens + dashboard
- Performance-related const constructor additions
</objective>

  <name>Task 1: Remote config + in-app update check</name>
  <files>
    supabase/migrations/009_remote_config.sql,
    lib/features/auth/screens/settings_placeholder.dart,
    lib/core/l10n/app_ar.arb,
    lib/core/l10n/app_en.arb
  </files>
  <action>
    **Part A — Supabase migration:**

    Create `supabase/migrations/009_remote_config.sql`:
    ```sql
    CREATE TABLE remote_config (
      key TEXT PRIMARY KEY,
      value TEXT NOT NULL,
      updated_at TIMESTAMPTZ DEFAULT NOW()
    );

    -- No RLS needed — read-only public config
    ALTER TABLE remote_config ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "Anyone can read config" ON remote_config FOR SELECT USING (true);

    -- Seed with initial values
    INSERT INTO remote_config (key, value) VALUES
      ('latest_version', '0.1.0'),
      ('download_url', ''),
      ('min_version', '0.1.0');
    ```

    **Part B — Settings screen update check:**

    In `settings_placeholder.dart`:
    - Add a version info row below the user card showing `AppConstants.appVersion`
    - On build: fetch remote config from Supabase (`supabase.from('remote_config').select()`)
    - Compare `latest_version` from remote with `AppConstants.appVersion`
    - Use simple string comparison (semver — split by '.', compare each part as int)
    - If newer version available: show an update card with:
      - Icon: Icons.system_update
      - Text: "Version X.Y.Z available"
      - FilledButton.tonal: "Download Update" → open download_url via url_launcher or just copy to clipboard
    - If download_url is empty: hide the download button and show only the version info (graceful for owner who hasn't uploaded yet) <!-- audit-added -->
    - If no update or fetch fails: show nothing (graceful degradation)
    - Use a Riverpod FutureProvider for the remote config check (not inline FutureBuilder — prevents repeated fetches on widget rebuild) <!-- audit-added -->

    Since url_launcher is not in dependencies, use `launchUrl` from the `url_launcher` package OR simply show the URL in a dialog for the user to copy. To avoid adding a new dependency, show the URL in a selectable Text dialog.

    **L10n strings (4 new):**
    - `appVersion` / AR: "إصدار التطبيق" / EN: "App Version"
    - `updateAvailable` / AR: "تحديث متاح" / EN: "Update Available"
    - `downloadUpdate` / AR: "تحميل التحديث" / EN: "Download Update"
    - `latestVersion` / AR: "لديك أحدث إصدار" / EN: "You have the latest version"
  </action>

    2. Set remote_config latest_version to "0.2.0" in Supabase → reload settings → update card appears
    3. Set remote_config latest_version to "0.1.0" → no update card
    4. Deploy migration: `supabase db push`
    5. `flutter analyze` — no errors
  </verify>

  <action>
    **Part A — RepaintBoundary on list cards:**

    Wrap each list item card widget in `RepaintBoundary` to isolate paint operations:
    - `order_list_screen.dart`: Wrap `_OrderCard(...)` return in `RepaintBoundary(child: Card(...))`
    - `payment_list_screen.dart`: Wrap `_PaymentCard(...)` return in `RepaintBoundary(child: Card(...))`
    - `package_list_screen.dart`: Wrap `_PackageLogCard(...)` return in `RepaintBoundary(child: Card(...))`

    Add RepaintBoundary at the Card level inside each widget's build method — wrap the outermost Card widget.

    **Part B — Dashboard list items:**

    In `owner_dashboard_screen.dart`:
    - Wrap each ListTile in the debtors section with RepaintBoundary
    - Wrap each ListTile in the package alerts section with RepaintBoundary

    **Part C — Const constructor audit:**

    Review and add `const` to widget constructors that are currently missing it:
    - Check all `SizedBox(height: N)` and `SizedBox(width: N)` — add const
    - Check `EdgeInsets.all/symmetric/only` — add const where used as constructor defaults
    - Check `Icon(...)` with only compile-time params — add const
    - Check `Divider()` — add const

    Focus on the files already being modified. Do NOT audit the entire codebase.

    Avoid: Adding RepaintBoundary to non-list widgets (overhead without benefit). Avoid Riverpod `select()` refactoring (too invasive for this plan). Avoid adding new packages for profiling.
  </action>
