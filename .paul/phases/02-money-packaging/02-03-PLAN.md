<objective>
## Goal
Driver can print order receipts and payment receipts via Bluetooth thermal printer. Arabic RTL text is handled via image rasterization. Printer connection is managed from the settings screen.

## Purpose
Paper receipts are essential for the wholesale field workflow — stores need a physical record of deliveries and payments. Drivers currently see receipts on screen but can't print them. This plan enables the Bluetooth printing that the business requires for daily operations.

## Output
- `lib/features/printing/` — print service, printer provider, printer setup screen
- Modified receipt preview with working Print button
- Modified payment list with print capability
- Settings screen with printer connection management
- Android Bluetooth permissions configured
</objective>

## Research: Package Selection
**Selected: `unified_esc_pos_printer`**
- Supports Bluetooth Classic, BLE, USB, Network via single PrinterManager API
- **Arabic RTL via `textRaster()`** — renders text with Flutter's text engine as image, prints as bitmap
- Solves the fundamental ESC/POS limitation: printers don't support RTL/Arabic glyph shaping natively
- Alternative `esc_pos_bluetooth` has known Arabic issues (prints Chinese characters)

## Research: Arabic Thermal Printing
- ESC/POS printers do NOT natively support Arabic RTL or glyph shaping
- Solution: Rasterize text to bitmap image, print as image
- `unified_esc_pos_printer.textRaster()` handles this automatically
- Receipt width: 58mm (32 chars) or 80mm (48 chars) — target 58mm (most common portable)
