<objective>
## Goal
Driver can collect cash payments from stores and record them. Each payment updates the store's credit balance. Orders also update credit balance (store owes more). Owner sees all payments and balances.

## Purpose
Financial accountability is the #1 business need after proving the core loop. Currently orders exist but money isn't tracked — the owner has no way to see what drivers have collected, what stores owe, or reconcile cash. This plan closes that gap.

## Output
- `lib/features/payments/` — repository, providers, and 2 screens
- New Supabase migration with RPC functions for atomic balance updates
- Modified order creation to update credit balance on new orders
- Modified driver shell (Payments tab wired) and owner shell (payments accessible)
- Updated l10n with payment-related Arabic strings
</objective>

## Supabase Schema Reference
payments: id(uuid), business_id(uuid), store_id(uuid), driver_id(uuid), amount(numeric >0),
  method(text default 'cash'), previous_balance(numeric), new_balance(numeric), created_at(timestamptz)

stores.credit_balance: NUMERIC NOT NULL DEFAULT 0
  - Positive = store owes us (debt from orders)
  - Negative = we owe store (overpayment, unlikely)

RLS: drivers can insert own payments (driver_id = auth.uid()), select own. Owners full access.
RLS: drivers can only SELECT stores, NOT UPDATE — need RPC function for atomic balance updates.

    **Supabase Migration** (NEW file: supabase/migrations/002_payment_functions.sql):
    Create two RPC functions for atomic balance updates (SECURITY DEFINER to bypass driver's read-only stores RLS):

    1. `create_payment(p_store_id UUID, p_amount NUMERIC, p_business_id UUID)` → RETURNS JSONB:
       - <!-- audit-added --> **Authorization check:** Verify p_business_id matches the caller's business_id from JWT: `IF p_business_id != (auth.jwt() -> 'user_metadata' ->> 'business_id')::UUID THEN RAISE EXCEPTION 'unauthorized';`
       - <!-- audit-added --> **Row lock:** `SELECT credit_balance INTO v_previous FROM stores WHERE id = p_store_id FOR UPDATE` (prevents race condition when two drivers pay same store simultaneously)
       - v_new := v_previous - p_amount
       - INSERT INTO payments (business_id, store_id, driver_id, amount, method, previous_balance, new_balance) VALUES (p_business_id, p_store_id, auth.uid(), p_amount, 'cash', v_previous, v_new)
       - UPDATE stores SET credit_balance = v_new WHERE id = p_store_id
       - RETURN jsonb with payment id, previous_balance, new_balance

    2. `update_store_balance_on_order(p_store_id UUID, p_order_total NUMERIC)` → RETURNS VOID:
       - <!-- audit-added --> **Row lock:** `UPDATE stores SET credit_balance = credit_balance + p_order_total WHERE id = p_store_id` (UPDATE is atomic, no explicit lock needed — credit_balance + p_order_total is safe)
       - This is called after successful order creation

    **IMPORTANT:** After writing the migration file, tell the user to run it on Supabase dashboard SQL Editor manually (we don't have CLI access). Include a reminder in the checkpoint.

    **PaymentRepository** (follow OrderRepository pattern):
    - Constructor takes SupabaseClient + businessId
    - `getAll({String? driverId})` — select payments with store join and driver join, ordered by created_at desc
      - Use `.select('*, stores(name), users!payments_driver_id_fkey(name)')`
      - Optional driverId filter (driver sees own, owner sees all)
    - `create({required String storeId, required double amount})`:
      - Call RPC function: `_client.rpc('create_payment', params: {...})`
      - Pass store_id, amount, business_id
      - Return the RPC result (jsonb with payment details)
    - `getByStore(String storeId)` — payments for a specific store, ordered by created_at desc

    **PaymentProvider** (follow order_provider.dart pattern):
    - `paymentRepositoryProvider` — Provider<PaymentRepository?> watching currentUserProvider
    - `paymentListProvider` — FutureProvider for driver's payments (filtered by driverId)
    - `allPaymentsProvider` — FutureProvider for owner (no driver filter)
