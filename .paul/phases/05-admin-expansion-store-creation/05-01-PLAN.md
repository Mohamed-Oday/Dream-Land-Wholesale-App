- Owner dashboard uses: `_SectionHeader`, `_EmptyState`, `_buildShimmerList` — private widgets that admin dashboard will implement its own versions of
- `ProductListScreen` has no role restrictions — FAB always visible

## Admin Dashboard Design
Admin dashboard-lite shows 3 sections (NO revenue KPIs, NO pending discounts, NO approve/reject):
1. **Recent Orders** — Last 10 orders with store name, driver name, total, timestamp
2. **Top Debtors** — Stores with highest credit balance (reuse `topDebtorsProvider`)
3. **Package Alerts** — Stores exceeding package threshold (reuse `packageAlertsProvider`)

    In `dashboard_repository.dart`, add method:
    ```dart
    Future<List<Map<String, dynamic>>> getRecentOrders({int limit = 10}) async {
      final result = await _client
          .from('orders')
          .select('*, stores(name, address), users!orders_driver_id_fkey(name)') <!-- audit-added: match existing OrderRepository join pattern -->
          .eq('business_id', _businessId)
          .order('created_at', ascending: false)
          .limit(limit);
      return List<Map<String, dynamic>>.from(result);
    }
    ```
    Uses same join pattern as OrderRepository.getAll(). Admin has `orders_admin_select` RLS.

    **Step 2: Add recentOrdersProvider to dashboard_provider.dart**

    ```dart
    final recentOrdersProvider = FutureProvider<List<Map<String, dynamic>>>((ref) async {
      final repo = ref.watch(dashboardRepositoryProvider);
      if (repo == null) return [];
      return repo.getRecentOrders();
    });
    ```
