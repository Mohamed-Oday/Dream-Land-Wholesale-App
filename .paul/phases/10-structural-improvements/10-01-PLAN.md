<objective>
## Goal
Add CHECK constraints, updated_at auto-timestamps, and cancellation audit trail columns to core tables — plus move the last inline RPC call into a repository.

## Purpose
**CHECK constraint:** `stock_on_hand` can currently go negative via race conditions. A database-level CHECK prevents corrupt inventory data.

**updated_at columns:** 4 core tables (orders, stores, products, users) lack modification timestamps. Without these, there's no way to know when a record was last changed — a basic audit requirement.

**Cancellation audit trail:** `cancel_order` sets `status = 'cancelled'` but doesn't record who cancelled or when. This gaps the audit trail for the most sensitive order state transition.

**Inline RPC cleanup:** One remaining screen (`stock_adjustment_screen.dart`) calls `Supabase.instance.client.rpc()` directly instead of going through the repository layer.

## Output
- Migration `018_schema_hardening.sql` (CHECK, updated_at triggers, cancellation columns, cancel_order update)
- Cleaned up stock adjustment screen using repository
</objective>
