## Goal
Replace the owner dashboard placeholder with a real-time metrics dashboard showing today's revenue, today's order count, top debtor stores, and package alerts — giving the owner immediate operational visibility.

## Purpose
The owner's core need is **real-time visibility into field operations**. The dashboard is the first screen they see after login and must answer: "How is today going? Who owes me money? Where are my packages?" This directly delivers the project's core value proposition.

## Output
- Supabase RPC function for package alert aggregation
- DashboardRepository + Riverpod providers for all dashboard metrics
- OwnerDashboardScreen with KPI cards, debtor list, package alerts
- Owner shell updated to use real dashboard instead of placeholder
</objective>

## UI/UX Design Specifications (from /ui-ux-pro-max)

### Dashboard Layout (RTL Arabic, Material 3)
- **KPI Cards:** Horizontal row of 2 cards (Revenue + Orders) at top
  - Large number (headlineMedium, fontFeatures: [FontFeature.tabularFigures()])
  - Label below in bodySmall
  - Card elevation: 0, filled tonal surface
  - Min height: 100dp for comfortable touch
- **Sections below:** "Top Debtors" and "Package Alerts" as titled list sections
  - Section header: titleMedium weight 600
  - List tiles: store name, balance amount, subtle trailing icon
  - Max 5 items per section, "See all" if more exist
- **Empty states:** Icon + descriptive text + suggested action per section
- **Loading:** Shimmer/skeleton placeholders matching card and list dimensions
- **Pull-to-refresh:** RefreshIndicator wrapping ScrollView

### Design Rules Applied
- Spacing: 8dp grid (padding 16, gaps 12-16)
- Touch targets: min 48dp height on list tiles
- Number formatting: DA currency with locale-aware separators
- RTL: All layout inherits from app-level Directionality
- Colors: Use theme colorScheme (primary, surface, surfaceContainerHighest, error)
- Contrast: 4.5:1+ for all text on surfaces
- Typography: Use theme textTheme roles (headlineMedium for KPI values, titleMedium for section headers, bodyMedium for list content)


```gherkin

And when all packages are returned, an empty state message is shown
```
