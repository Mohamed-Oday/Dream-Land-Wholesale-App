<objective>
## Goal
Enable drivers to request order discounts during creation and give the owner approve/reject control on the dashboard, with automatic rejection after 3 minutes to prevent field blocking.

## Purpose
Drivers need flexibility to offer discounts to stores (bulk deals, damaged goods, loyalty). The owner needs oversight — every discount must be approved or auto-expires. This closes the last operational control gap: the owner sees orders, payments, packages, driver locations, and now controls pricing.

## Output
- Supabase RPCs for approve/reject/auto-reject expired discounts
- Discount input on create order screen with pending status
- Discount status indicators on order list and receipt
- "Pending Discounts" section on owner dashboard with approve/reject actions
- 3-minute auto-reject via server-side RPC on dashboard load
</objective>

## UI/UX Design Specifications (from /ui-ux-pro-max)

### Discount Input (Create Order Screen — Driver)
- Collapsible section after order summary, labeled "خصم" with percent icon
- TextFormField: number keyboard, DA currency suffix, min 0, max <= subtotal
- When discount > 0: info chip below field "يتطلب موافقة المالك" in warning color
- Updated total displayed: shows original total crossed out, discounted total below
- Validation: discount cannot exceed subtotal

### Discount Status Badge (Order List)
- Chip next to existing order status chip
- States: pending (warning/orange), approved (success/green), rejected (error/red)
- Only shown when discount_status != 'none'

### Pending Discounts Section (Owner Dashboard)
- New section ABOVE "Top Debtors" (more urgent)
- Section header with warning icon + count badge
- Each pending item: Card with store name, driver name, discount amount, time remaining
- Two action buttons per card: Approve (FilledButton, primary) + Reject (OutlinedButton, error)
- Confirmation dialog before approve/reject
- Cards disappear after action (animate out)
- Empty state: "لا توجد خصومات معلقة" with checkmark icon

### Design Rules Applied
- Touch targets: approve/reject buttons min 48dp
- Spacing: 8dp grid
- Colors: warning for pending, success for approved, error for rejected — always with icon+text (not color alone)
- Confirmation: destructive action (reject) requires confirmation dialog
- Timeout display: show "remaining X:XX" countdown text in bodySmall

    **1. Create Supabase RPC functions** (`006_discount_functions.sql`):

    `approve_discount(p_order_id UUID, p_approved_by UUID, p_business_id UUID)`:
    - Authorization: verify caller's business_id matches param
    - Verify order exists and discount_status = 'pending' — if NOT pending, RAISE EXCEPTION 'discount_already_processed' (prevents stale UI actions on already-approved/rejected/auto-rejected discounts) <!-- audit-added -->
    - UPDATE orders SET discount_status = 'approved', discount_approved_by = p_approved_by WHERE id = p_order_id
    - Return updated order data
    - SECURITY DEFINER with GRANT EXECUTE

    `reject_discount(p_order_id UUID, p_business_id UUID)`:
    - Authorization: verify caller's business_id matches param
    - Verify order exists and discount_status = 'pending' — if NOT pending, RAISE EXCEPTION 'discount_already_processed' <!-- audit-added -->
    - Get order's discount amount and store_id
    - UPDATE orders SET discount_status = 'rejected', total = subtotal + tax_amount WHERE id = p_order_id
    - UPDATE stores SET credit_balance = credit_balance + discount WHERE id = store_id (add discount back to store debt)
    - Both in single transaction
    - Return updated order data
    - SECURITY DEFINER with GRANT EXECUTE

    `reject_expired_discounts(p_business_id UUID)`:
    - Authorization: verify caller's business_id
    - Find all orders WHERE business_id = p_business_id AND discount_status = 'pending' AND created_at < now() - interval '3 minutes'
    - For each: update discount_status = 'rejected', recalculate total, adjust store credit_balance
    - Return count of rejected orders
