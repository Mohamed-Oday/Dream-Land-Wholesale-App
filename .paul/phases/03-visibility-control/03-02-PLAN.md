<objective>
## Goal
Enable driver GPS tracking with on-duty toggle and show live driver positions on an OpenStreetMap-based map for the owner — giving the owner real-time visibility into where drivers are in the field.

## Purpose
The owner needs to know where drivers are at any given moment. "Which driver is closest to store X? Is the driver actually visiting stores or sitting idle?" This plan delivers the location tracking infrastructure and the visual map that answers these questions.

## Output
- Supabase RPC for fetching latest driver positions with names
- LocationService for GPS tracking with 30s broadcast interval
- LocationRepository for inserting positions to Supabase
- On-duty toggle banner in driver shell (persistent across all tabs)
- Live driver map replacing owner shell Map placeholder
- flutter_map + geolocator package integration
</objective>

## UI/UX Design Specifications (from /ui-ux-pro-max)

### On-Duty Toggle (Driver Shell)
- Persistent banner between body and bottom nav — visible on ALL tabs
- Height: 48dp, horizontally expanded
- On-duty state: filled primary color background, white text "في الخدمة", location icon pulsing
- Off-duty state: surfaceContainerHighest background, onSurfaceVariant text "خارج الخدمة"
- Toggle via tap on the entire banner (48dp touch target met)
- State transition: 200ms crossfade animation

### Live Map (Owner)
- Full-screen flutter_map filling the body below AppBar
- OpenStreetMap tile layer (no API key, free)
- Default center: Algeria (lat 36.7, lng 3.0), zoom 12
- Driver markers: CircleAvatar with driver initial letter, primary color, 40dp diameter
- Tap marker → BottomSheet with: driver name, "last seen X min ago", close button
- FAB for refresh (bottom-right, above bottom nav safe area)
- Empty state overlay when no active drivers: icon + "لا يوجد سائقين في الخدمة"
- Loading: shimmer/gray map placeholder while tiles load

### Design Rules Applied
- Touch targets: map markers 40dp (extended to 48dp via GestureDetector padding)
- Spacing: 8dp grid
- Colors: theme colorScheme tokens only
- RTL: text in bottom sheet right-aligned, FAB respects directionality
- Map gestures: pinch-zoom, pan — no conflict with system gestures
- Safe areas: map extends edge-to-edge, controls respect safe area insets

```gherkin
Given the driver is on-duty
When 30 seconds elapse
Then the driver's current GPS position is inserted into Supabase driver_locations

And broadcasting stops when the driver goes off-duty
```

    **3. Create Supabase RPC** (`005_location_functions.sql`):
    `get_latest_driver_locations(p_business_id UUID)` — returns latest position per active driver:
    - DISTINCT ON (dl.driver_id) from driver_locations
    - JOIN users ON users.id = dl.driver_id to get driver name
    - Filter: dl.business_id = p_business_id AND dl.timestamp > now() - interval '1 hour' (only recent) AND u.active = true (exclude deactivated drivers) <!-- audit-added -->
    - ORDER BY dl.driver_id, dl.timestamp DESC
    - Returns: driver_id, driver_name, lat, lng, timestamp
    - SECURITY DEFINER with business_id authorization check (same pattern as 004)
    - GRANT EXECUTE to authenticated

    **4. Create LocationRepository** (`location_repository.dart`):
    - Constructor: SupabaseClient + businessId (matches existing repos)
    - `insertPosition({required String driverId, required double lat, required double lng})` → void
      Insert into driver_locations with business_id, driver_id, lat, lng
    - `getLatestDriverLocations()` → `List<Map<String, dynamic>>`
      Call RPC `get_latest_driver_locations` with business_id

    **5. Create LocationService** (`location_service.dart`):
    - Uses `geolocator` package for GPS position
    - `Future<bool> checkAndRequestPermission()` — first checks if location services are enabled via `Geolocator.isLocationServiceEnabled()` (GPS on/off at device level). If disabled, return false with a message to enable GPS. Then checks location permission, requests if not granted via `Geolocator.requestPermission()`. Returns true only if both services enabled AND permission granted. <!-- audit-added: must check GPS service state, not just permission -->
    - `Stream<Position> startTracking({Duration interval = const Duration(seconds: 30)})` — starts periodic GPS reads. Uses `Geolocator.getPositionStream()` with `LocationSettings(distanceFilter: 10, timeLimit: interval)`. Falls back to `Timer.periodic` + `Geolocator.getCurrentPosition()` if stream not suitable.
    - `void stopTracking()` — cancels stream/timer subscription

    **6. Create location providers** (`location_provider.dart`):
    - `locationRepositoryProvider` — Provider creating LocationRepository from SupabaseClient + currentUser.businessId
    - `locationServiceProvider` — Provider creating LocationService singleton
    - `isOnDutyProvider` — StateProvider<bool> defaulting to false
    - `driverLocationsProvider` — FutureProvider calling getLatestDriverLocations() (for owner map)
    - Side-effect: when isOnDutyProvider changes to true, start tracking and insert positions. When false, stop tracking. Handle this in the driver shell widget, not in the provider itself (providers are declarative).

    **7. Update driver_shell.dart — add on-duty toggle banner:**
    - Add a persistent banner widget between body content and bottom NavigationBar
    - Banner structure: InkWell wrapping a Container(height: 48), Row with Icon + Text + Switch
    - On-duty (true): backgroundColor = colorScheme.primaryContainer, text "في الخدمة" (l10n.onDuty), leading Icon(Icons.location_on, color: primary)
    - Off-duty (false): backgroundColor = colorScheme.surfaceContainerHighest, text "خارج الخدمة" (l10n.offDuty), leading Icon(Icons.location_off)
    - Toggling on: check permission first via LocationService.checkAndRequestPermission(). If denied, show SnackBar with message and revert toggle. If granted, set isOnDutyProvider to true and start location timer.
    - Toggling off: set isOnDutyProvider to false, stop location timer.
    - Use ConsumerStatefulWidget to manage StreamSubscription for location updates.
    - On each position update, call locationRepository.insertPosition() wrapped in try-catch. On failure (network unavailable), silently skip — do NOT show error toast every 30s. Log if needed but don't disrupt the driver. <!-- audit-added: silent failure handling for periodic inserts -->
    - Dispose subscription in dispose().

    Avoid:
    - Do NOT use background isolates or workmanager — simple foreground timer is sufficient for MVP
    - Do NOT add a separate Settings toggle — the banner is the single source of truth
    - Do NOT modify individual screen AppBars
    - Do NOT persist on-duty state across app restarts (default to off-duty on launch)
