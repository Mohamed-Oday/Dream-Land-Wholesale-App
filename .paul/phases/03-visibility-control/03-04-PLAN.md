<objective>
## Goal
Create a store detail drill-down screen (orders, payments, packages for a specific store) and admin driver management (list, create, deactivate drivers) — completing Phase 3's visibility and control features.

## Purpose
The owner needs to answer "What's the full picture for this store?" by tapping a store name anywhere in the app. The admin needs to manage the driver workforce without asking the owner. These are the final operational gaps in the app.

## Output
- StoreDetailScreen with store info, recent orders, payments, and package balances
- Store list and dashboard debtors navigate to detail on tap
- UserManagementScreen reusable for both owner and admin
- Owner can create admins + drivers (5th tab: "المستخدمين")
- Admin can create drivers only (existing drivers tab)
- Admin shell + owner shell updated with real user management
</objective>

## UI/UX Design Specifications (from /ui-ux-pro-max)

### Store Detail Screen
- AppBar: store name as title, edit icon action (navigates to StoreFormScreen)
- **Store Info Card:** Card at top with address, phone, contact person, credit_balance (red if debt)
- **Scrollable body** with 3 sections (not tabs — simpler for mobile):
  - "الطلبات الأخيرة" (Recent Orders) — last 10, same card style as order_list_screen
  - "المدفوعات الأخيرة" (Recent Payments) — last 10, same tile style
  - "رصيد العبوات" (Package Balances) — per-product current balances
- Each section: header + list, with "عرض الكل" if more than 10
- Empty states per section

### Driver List Screen (Admin)
- AppBar: "السائقين" (Drivers), no special actions
- ListView of drivers: name, username, active badge (green/red)
- FAB to add new driver
- Long-press or trailing button to deactivate/activate
- Confirmation dialog before deactivation
- Create driver: AlertDialog with form (name, username, password) — same pattern as old owner_shell dialog

### Design Rules Applied
- Touch targets: 48dp on list items
- Spacing: 8dp grid, 16dp padding
- Back navigation: predictable, preserves scroll state
- Empty states with icon + message per section
- Confirmation before destructive actions (deactivate driver)
- Colors: theme colorScheme tokens only
